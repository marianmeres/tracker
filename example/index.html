@marianmeres/tracker — vanilla example

Open the browser console. Every track() call is logged (via debug: true); every batch flush is logged by the transport itself. Default flush is every 5s or when 10 events queue up.

Strategy 1: inline onclick

The tracker is exposed as window.tracker, so handlers can call it directly.

Strategy 2: data-* attributes

A single document-level click listener reads data-track (event name) and the optional data-track-payload (JSON). No per-element JS — just markup.
